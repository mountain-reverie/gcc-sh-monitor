gcc-sh-monitor

Continuous testing of GCC's SuperH backend — correctness, code size, and cross-architecture comparison on every upstream commit.

What's measured

Each ci-full run rebuilds GCC HEAD as a cross-toolchain for sh4-linux-gnu, arm-linux-gnueabihf, and i686-linux-gnu, then runs the suites below. Multi-arch metrics surface SH4-specific issues vs general GCC behaviour: if all three arches change, it's a generic regression; if only sh4 moves, it's an SH backend issue.

Correctness (sh4 only)

  • dejagnu — GCC's own sh.exp testsuite run under qemu-sh4-static via a custom board file. See the failing tests panel below.
  • LRA differential — a curated corpus of LRA-sensitive cases, compiled by both trunk GCC and Debian's pre-LRA GCC 14.2; cases where trunk does worse are surfaced as regressions. Detail: /lra/.

Code size (sh4 / arm / x86)

  • CSiBE — 16 real-world C subprojects (bzip2, jpeg, libpng, zlib, flex, …). Sum of .text + .rodata + .data per project at -Os and -O2.
  • CoreMark 1.0 — EEMBC benchmark, built with published-score flags. Tracks .text and total binary size.
  • BusyBox (glibc-static) — defconfig built against Debian's glibc, static-linked. 10-applet smoke under qemu.
  • BusyBox+musl — same defconfig, but linked against our-built musl. Libc-equalised comparison — isolates source codegen from libc tuning differences. (Alpine Linux ships this configuration.)
  • BusyBox+musl FDPIC (sh4 only) — the same build under the ELF FDPIC ABI (musl + libgcc compiled -mfdpic). glibc has no SH-FDPIC ABI, so this is musl-only. Serves as a position-independent reference ceiling for the shared constant pool work.
  • musl libc.so — musl built from source per arch, .text of the shared library measured. Static hello-world smoke validates the build actually links and runs.

Trend charts

All metrics over time — multi-arch metrics render as 3-line charts (sh4 blue / arm green / x86 orange). LRA differential table — per-case outcomes. Raw data as JSON.
